# 博客系统 — 完整 DDD 设计包 + 用户故事（可导入项目管理）

## 一、用户故事表（Jira / Trello / GitHub Projects 可用）

| ID | 标题 | 描述 | 验收标准 | 估算 (天) |
|----|------|------|----------|-----------|
| US-0-1 | 初始化项目骨架 | 创建后端项目（Spring Boot / NestJS）、前端项目（Next.js + Tailwind），配置 Lint/Prettier/Tailwind | 项目可运行，首页显示 Hello World | 1 |
| US-0-2 | 用户鉴权基础 | 用户注册/登录/登出，JWT 或 Cookie 保存用户状态，保护 Admin 路由 | 用户可登录，未登录用户无法访问后台管理页面 | 2 |
| US-1-1 | 文章聚合基础 | 设计 Article 实体，实现 Repository 接口，API: 创建/获取文章 | 前端可创建文章并在列表中看到标题 | 3 |
| US-1-2 | 文章列表与分页 | 后端 API 支持分页查询文章，前端展示文章列表页 | 首页能分页显示文章列表 | 2 |
| US-1-3 | 文章详情页 | API 提供通过 slug 或 id 获取文章详情，前端渲染 Markdown + 封面图 | 点击列表文章跳转详情页，正确渲染内容 | 2 |
| US-1-4 | 文章发布状态管理 | ArticleStatus: DRAFT/PUBLISHED/ARCHIVED，API 发布文章，前端管理草稿与发布状态 | 文章可保存草稿，管理员可发布 | 2 |
| US-1-5 | 标签和分类 | 创建 Tag/Category 实体，API 支持关联文章，前端显示标签/分类 | 文章可以选择标签和分类，前端展示正确 | 2 |
| US-2-1 | 评论聚合基础 | Comment 实体 + Repository，API: 提交评论/获取评论列表，前端 CommentList/CommentForm 组件 | 用户可在文章详情页发表评论并看到列表 | 2 |
| US-2-2 | 评论审核 | Comment 状态: PENDING/APPROVED/REJECTED，Admin 后台审核评论 | 管理员可以批准或拒绝评论，前端只显示 APPROVED 评论 | 2 |
| US-3-1 | 文章封面 & 图片上传 | MediaFile 实体 + Repository，上传接口，前端选择封面图和文章中插图 | 上传图片后可在文章列表和详情页显示 | 3 |
| US-4-1 | 文章搜索 | Search BC: 支持关键词/标签搜索，API 返回匹配文章，前端搜索栏和结果页面 | 用户可以搜索文章，结果按相关度或发布时间排序 | 2 |
| US-4-2 | 文章归档 | 按月份/年份整理文章，API 提供归档数据，前端展示归档页 | 用户可以按月查看文章列表 | 2 |
| US-5-1 | 文章浏览统计 | PV/UV 统计，每次访问计数，后端保存统计数据，前端显示阅读量 | 文章详情页显示阅读次数并随访问增加 | 2 |
| US-5-2 | 评论通知 | 评论审核后发通知（邮件或 Webhook，可模拟），前端 Admin 可触发测试通知 | 提交评论并审核后可收到通知 | 2 |
| US-6-1 | 用户个人信息管理 | 用户可修改头像、昵称、个人简介 | 修改信息后前端立即生效 | 1 |
| US-6-2 | 暗/亮主题切换 | 前端全局主题切换，配置存储在 Configuration BC | 用户可以切换主题并保存偏好 | 1 |
| US-6-3 | RSS / 订阅功能 | 提供 RSS Feed，前端可订阅文章更新 | RSS 正确生成文章列表 | 2 |

## 二、开发顺序建议

```
阶段0：US-0-1 → US-0-2
阶段1：US-1-1 → US-1-2 → US-1-3 → US-1-4 → US-1-5
阶段2：US-2-1 → US-2-2
阶段3：US-3-1
阶段4：US-4-1 → US-4-2
阶段5：US-5-1 → US-5-2
阶段6：US-6-1 → US-6-2 → US-6-3
```

> 每个用户故事工作量 ≤ 1 周，可用于迭代式开发。

---

## 三、使用方法
1. 可直接将此表格导入 Jira / Trello / GitHub Projects。  
2. 每条用户故事对应一个任务卡，填写标题、描述、验收标准和预计工作天数。  
3. 按顺序迭代完成，每阶段完成后可进行回顾和优化。
